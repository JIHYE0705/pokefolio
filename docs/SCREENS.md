# Pokefolio 화면 명세

이 문서는 [제품 비전](PRODUCT_VISION.md)을 390px 모바일 화면의 정보 우선순위로 번역한다. 모든 기능은 기획 단계이며 실제 치수와 문구는 승인된 목업과 접근성 검증 후 확정한다.

공통으로 loading, empty, error, success를 지원하고 하단 내비게이션은 safe area 위에 배치한다. 카드 이미지와 사용자의 기록이 UI chrome보다 먼저 보여야 한다.

## 공통 이동 구조

```text
Home ─┬─ Collection ─ Card Detail ─ Wishlist
      ├─ Collection Journal ─ Create / Detail
      ├─ Binder ─ Binder Detail
      └─ Memories / Collection Journey

Keeper → 관련 콘텐츠 안에서만 짧게 노출
Trade → Wishlist 또는 Duplicates의 후속 단계
```

공식 기록 용어는 `Collection Journal`이다. 현재 Frontend의 `/opening-logs`는 이전 mock route이며 이번 문서 작업에서 코드나 경로를 변경하지 않는다.

## Home

**사용자가 느껴야 하는 감정:** 다시 보고 싶은 반가움, 이어서 꾸미고 싶은 가벼운 기대, 쌓여가는 여정의 뿌듯함.

**화면의 핵심 목적:** 사용자가 지금 보고 싶거나 이어서 하고 싶은 한 가지를 빠르게 발견한다. Home은 컬렉션 통계 대시보드가 아니다.

**첫 번째로 보여야 할 콘텐츠:** 가장 최근에 편집한 미완성 Binder와 `이어서 꾸미기`. 사용자가 기록 직후 돌아왔다면 방금 남긴 Collection Journal을 우선할 수 있다.

**콘텐츠 우선순위**

1. 이어서 꾸밀 Binder
2. 최근의 수집 기억
3. 최근 추가한 카드
4. 과거의 오늘
5. Keeper가 발견한 컬렉션 변화
6. 오늘 할 수 있는 간단한 행동

**핵심 CTA:** `이어서 꾸미기`. 이어갈 Binder가 없으면 `오늘의 수집 기록하기`를 대신 사용한다.

**보조 CTA:** 최근 기억 열기, 최근 카드 보기, Collection Journal 작성.

**표시하면 안 되는 정보:** 동일한 크기의 총 카드·중복·위시리스트·시세 통계 카드, 투자 가치, 복잡한 그래프, 여러 추천 박스, 경쟁하는 CTA, 근거 없는 AI 배지.

**상태**

- Loading: Binder preview, 최근 기억과 카드 행의 콘텐츠 형태를 유지하는 절제된 skeleton을 보여준다.
- Empty: 통계 대신 `첫 수집 기록 남기기` 한 가지 행동과 카드 또는 순간을 기록할 수 있다는 짧은 안내를 제공한다.
- Error: 불러오지 못한 영역을 구분하고 `다시 불러오기`를 제공한다. 정상적으로 불러온 다른 콘텐츠는 숨기지 않는다.
- Success: Binder 또는 최근 기억이 첫 화면에서 가장 먼저 읽히고 Keeper는 실제 변화가 있을 때만 한 문장으로 나타난다.

```text
┌────────────────────────────┐
│ Pokefolio                  │
│ 안녕하세요, 지혜님.         │
│                            │
│ 이어서 꾸미기               │
│ 봄빛 바인더          7 / 9 │
│ [card] [card] [card]       │
│ [       이어서 꾸미기      ] │
│                            │
│ 최근의 기억                 │
│ 7월 16일 · 첫 카드샵 방문   │
│ [photo] "여행 끝에 만났다." │
│                            │
│ 최근 추가                   │
│ [card] [card] [card]   ›   │
│                            │
│ 1년 전 오늘                 │
│ 포켓몬 151을 처음 개봉했어요.│
│                            │
│ Keeper                     │
│ 따뜻한 색감이 늘고 있어요.  │
│                            │
│ Home Collection Journal Binder│
```

## Collection

**사용자가 느껴야 하는 감정:** 모아 온 카드를 바라보는 뿌듯함과 자신의 취향을 발견하는 즐거움.

**화면의 핵심 목적:** 보유 카드를 감상하고 찾는 동시에 최근 카드, 최애, 포켓몬·테마·확장팩·색감별 모음에서 수집 성향을 탐색한다.

**첫 번째로 보여야 할 콘텐츠:** 실제 카드 이미지. 기본 진입에서는 최근 추가 카드를 먼저 보여주고 검색 중일 때는 검색 결과를 우선한다.

**핵심 CTA:** `카드 추가`.

**보조 CTA:** 검색, 필터·정렬, 최애 또는 테마 모음 열기, 중복 확인, Card Detail 열기.

**표시하면 안 되는 정보:** 카드 이미지보다 큰 필터·통계·배지, 시세 합계, 수익률, 화면을 채우기 위한 취향 점수, 반복되는 수량 타일과 과도한 분류 메뉴.

**상태**

- Loading: 검색과 현재 보기 선택을 유지하고 카드 비율의 skeleton을 표시한다.
- Empty: 전체 Collection이 비었으면 `첫 카드 추가`, 검색 결과만 없으면 검색어 수정 또는 필터 초기화를 제안한다.
- Error: 검색어와 필터를 보존하고 카드 영역에서 재시도한다.
- Success: 카드 이미지, 이름, 획득 시점 또는 짧은 맥락을 먼저 보여주고 수량·중복은 보조 정보로 둔다.

```text
┌────────────────────────────┐
│ Collection              ＋ │
│ [ 카드명·세트·번호 검색   ] │
│                            │
│ 최근 추가              모두 ›│
│ [ card ] [ card ] [ card ] │
│                            │
│ 나의 모음                   │
│ 최애 카드 · 봄빛 · 물 타입  │
│                            │
│ 모든 카드        [필터] [정렬]│
│ [ card ]       [ card ]    │
│ 이름 · 7/16    이름 · ×2    │
│ [ card ]       [ card ]    │
│                            │
│ Home Collection Journal Binder│
└────────────────────────────┘
```

## Collection Journal

**사용자가 느껴야 하는 감정:** 그날의 설렘을 부담 없이 남기는 만족감과 시간이 지난 뒤 장면을 다시 만나는 반가움.

**화면의 핵심 목적:** 팩·박스 개봉에 한정하지 않고 카드를 얻거나 수집을 즐긴 순간을 카드, 사진과 짧은 이야기로 기록한다.

**첫 번째로 보여야 할 콘텐츠:** 최신 기록의 오늘의 카드 또는 사진, 날짜와 한 줄 기록. 작성 화면에서는 진입 경로에서 설정된 기록 유형이 먼저 보인다.

**핵심 CTA:** 목록의 `새 기록` 또는 작성 화면의 `기억 남기기`.

**보조 CTA:** 카드 추가, 오늘의 카드 선택, 사진 추가, 기록 상세·편집.

**표시하면 안 되는 정보:** 모든 항목을 필수로 만드는 긴 form, 구매 금액과 별점을 앞세운 소비 평가, 자동 생성된 감정 단정, 팩 개봉만 가능한 유형, 여러 저장 CTA.

**포함하는 기록 유형:** 팩·박스 개봉, 낱장 구매, 카드샵·팝업스토어 방문, 선물, 교환, 여행 중 구매, 특별한 카드와 연결된 개인적인 기억.

**빠른 기록 필드**

| 필드 | 필수 여부 | 기본값·규칙 |
| --- | --- | --- |
| 기록 유형 | 필수 | 진입 경로에서 설정 |
| 날짜·시간 | 자동 | 현재 시각 |
| 획득 카드 | 선택 | 없음 |
| 오늘의 카드 | 선택 | 획득 카드 중 선택 |
| 한 줄 기록 | 선택 | 없음 |
| 사진 | 선택 | 없음 |

기록 유형은 진입 경로에서 기본 설정하고 사용자가 확인하거나 변경한다. 날짜·시간은 화면 입력을 늘리지 않고 저장 시 현재 시각을 사용한다. 획득 카드가 없으면 오늘의 카드를 선택하지 않으며, 기록 저장이 Collection 수량을 자동 변경할지는 데이터·API 계약 확정 전 Open Question으로 둔다.

**상태**

- Loading: 최신 기록의 날짜·대표 이미지·문장 형태를 유지한다. 작성 중 카드 검색만 지연되면 입력값을 그대로 둔다.
- Empty: `첫 수집 기록 남기기`와 기록할 수 있는 경험 예시 한두 개만 제공한다.
- Error: 목록은 재시도를 제공하고 작성 실패는 모든 입력과 선택한 카드를 보존하며 필드 가까이에 원인을 연결한다.
- Success: 최신순 기록에서 카드나 사진과 짧은 이야기를 함께 보여준다. 저장 뒤에는 상세로 이동하고 간결한 완료 피드백을 제공한다.

```text
┌────────────────────────────┐
│ Collection Journal      ＋ │
│                            │
│ JUL 16                     │
│ [photo] 첫 카드샵 방문      │
│         여행 끝에 만난 카드 │
│         [card] [card]      │
│ ────────────────────────── │
│ JUL 02                     │
│ [card]  선물 받은 피카츄    │
│         함께여서 더 좋았다. │
│                            │
│ Home Collection Journal Binder│
└────────────────────────────┘
```

```text
┌────────────────────────────┐
│ 취소       오늘의 순간      │
│                            │
│ [ 기록 유형              › ]│
│                            │
│ 어떤 카드를 만났나요?       │
│ [         카드 추가        ]│
```

## Binder

**사용자가 느껴야 하는 감정:** 자신의 취향과 이야기를 직접 구성하는 즐거움, 채워지는 페이지를 바라보는 뿌듯함.

**화면의 핵심 목적:** 보유 카드를 슬롯에 저장하는 데 그치지 않고 계절, 색감, 포켓몬, 장소와 추억 같은 테마로 자신만의 이야기를 표현한다.

**첫 번째로 보여야 할 콘텐츠:** Binder 목록에서는 대표 카드와 3×3 페이지 preview, 상세에서는 실제 카드가 배치된 페이지 자체.

**핵심 CTA:** 목록에서는 `새 Binder`, 상세에서는 선택한 빈 슬롯의 `카드 추가`.

**보조 CTA:** `이어서 꾸미기`, 페이지 이동, 이름·짧은 설명·대표 카드·테마 편집, 카드 교체·비우기.

**표시하면 안 되는 정보:** 페이지보다 큰 관리 패널, 채운 슬롯 수만 강조한 성과 지표, 카드 이미지를 가리는 장식과 컨트롤, drag만 가능한 조작, 테마를 점수화한 AI 분석.

**테마 예시:** 봄, 바다, 노을, 분홍색, 귀여운 포켓몬, 여행, 특정 타입·포켓몬·일러스트레이터, 특별한 추억.

**상태**

- Loading: 대표 카드와 3×3 페이지 비율을 유지하는 skeleton을 보여준다.
- Empty: Binder가 없으면 `첫 Binder 만들기`, 빈 Binder 상세에서는 첫 슬롯의 `카드 추가`를 제공한다.
- Error: 목록은 재시도하고 상세는 현재 페이지 번호를 유지한다. 저장 실패 시 기존 배치를 화면에서 잃지 않는다.
- Success: 이름, 짧은 설명과 테마를 조용히 배치하고 카드 이미지와 페이지를 화면의 주인공으로 둔다.

```text
┌────────────────────────────┐
│ Binder                  ＋ │
│                            │
│ 봄빛 바인더                 │
│ 따뜻한 색의 봄 카드         │
│ [card] [card] [card]       │
│ [card] [card] [card]  7/9  │
│ [       이어서 꾸미기      ] │
│                            │
│ 여행에서 만난 카드          │
│ [card] [card] [  + ]       │
│                            │
│ Home Collection Journal Binder│
└────────────────────────────┘
```

```text
┌────────────────────────────┐
│ ‹ Binder   봄빛 바인더   ···│
│ 따뜻한 색의 봄 카드         │
│ Page 1 of 2          ‹  ›  │
│ [card] [card] [  + ]       │
│ [card] [card] [card]       │
│ [card] [  + ] [  + ]       │
│                            │
│ [       페이지 추가        ] │
└────────────────────────────┘
```

## 보조 화면

### Card Detail

큰 카드 이미지와 이름·세트·번호를 먼저 보여준다. 보유 수량, 상태, 메모, 연결된 Collection Journal과 Binder를 그다음에 둔다. 핵심 CTA는 미보유 카드의 `Collection에 추가`, 보유 카드의 `수집 기억 보기`이며 Wishlist와 Binder 배치는 보조 행동이다.

Loading은 카드 비율을 유지하고, 미보유 상태는 Empty가 아니라 명확한 보유 상태로 표현한다. Error는 재시도와 목록 복귀를 제공하며 Success는 카드와 사용자의 기억을 카탈로그 정보보다 우선한다.

### Wishlist

원하는 카드의 이미지를 중심으로 보여주고 우선순위·보유 여부는 보조 정보로 둔다. 핵심 CTA는 카드 상세 열기이며 제거와 중복 확인은 보조 행동이다. 시세와 구매 권유는 표시하지 않는다.

Loading은 카드 비율을 유지하고, Empty는 Card Detail에서 추가하는 방법을 안내한다. Error는 검색·필터를 보존하고, Success는 원하는 카드와 현재 Collection의 관계를 보여준다.

### Memories와 Collection Journey

Memories는 과거의 오늘, 이번 달의 기억, 첫 등록 카드, 여행에서 얻은 카드와 처음 완성한 Binder처럼 기존 기록을 자연스럽게 다시 보여주는 경험이다. Collection Journey는 월별·연도별 변화, 기억에 남는 기록, 완성한 Binder와 색감·테마 변화를 시간 흐름으로 보여준다.

둘 다 MVP의 독립 화면으로 확정하지 않는다. 먼저 Home과 Collection의 작은 섹션으로 가치를 검증한다. 숫자를 목표나 성과 압박으로 표현하지 않는다.

### Trade

Trade는 Wishlist와 Duplicates를 연결하는 후속 단계이며 MVP에서는 구현하지 않는다. 시세, 결제, 배송과 거래 중재를 약속하지 않고 현재 가능한 카드 정리만 안내한다.
