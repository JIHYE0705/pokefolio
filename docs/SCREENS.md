# Pokefolio 화면 명세

모든 와이어프레임은 390px 모바일 세로 화면의 구조를 단순화한 것이다. 실제 치수와 문구는 목업·접근성 검증 후 확정한다. 공통으로 loading, empty, error, success를 지원하며 하단 내비게이션은 safe area 위에 배치한다.

## 공통 이동 구조

```text
Home ─┬─ Collection ─ Card Detail ─ Wishlist
      ├─ Binder List ─ Binder Detail
      └─ Opening Log List ─ Create / Detail

Trade Coming Soon ← Home 또는 Wishlist의 보조 진입점
```

## Home

**목적:** 컬렉션 상태와 최근 기록을 빠르게 파악하고 다음 행동으로 이동한다.

**구성:** 인사/제목, 컬렉션 요약, 최근 추가 카드, 최근 개봉일기, 위시리스트 링크.

**핵심 액션:** 컬렉션 보기, 개봉일기 작성, 최근 카드·기록 열기.

**상태**

- Loading: 요약과 카드 행의 skeleton을 표시한다.
- Empty: 첫 카드 추가와 첫 개봉일기 중 우선 행동을 안내한다.
- Error: 불러오지 못한 영역과 전체 재시도를 제공한다.
- Success: 요약보다 카드 이미지와 최근 기록을 먼저 탐색할 수 있다.

**이동:** 하단 탭과 각 요약 카드에서 Collection, Binder, Opening Log로 이동한다.

```text
┌──────────────────────────┐
│ Pokefolio          ♡     │
│ 나의 수집 앨범            │
│ ┌──────────────────────┐ │
│ │ 보유 카드 128 · 중복 12│ │
│ └──────────────────────┘ │
│ 최근 추가                 │
│ [card] [card] [card] →   │
│ 최근 개봉일기             │
│ [7/16 세트명 · 5장]       │
│                          │
│ Home Collection Binder Log│
└──────────────────────────┘
```

## Collection

**목적:** 보유 카드를 검색·필터링하고 수량과 중복 여부를 확인한다.

**구성:** 제목, 검색, 필터/정렬, 결과 수, 카드 grid.

**핵심 액션:** 검색, 필터 변경, 카드 상세 열기, 카드 추가.

**상태**

- Loading: 검색은 유지하고 카드 비율의 skeleton grid를 표시한다.
- Empty: 컬렉션이 비었으면 카드 추가, 검색 결과가 없으면 조건 초기화를 제안한다.
- Error: 검색어와 필터를 보존한 채 재시도한다.
- Success: 카드 이미지, 이름, 세트, 보유 수량을 보여준다.

**이동:** 카드 선택 시 Card Detail, `+` 시 카드 검색/추가 흐름으로 이동한다.

```text
┌──────────────────────────┐
│ Collection           ＋  │
│ [ 카드명·세트·번호 검색 ] │
│ [필터] [정렬]       128장 │
│ [ card ]   [ card ]      │
│ 이름 · ×2  이름 · ×1      │
│ [ card ]   [ card ]      │
│ 이름 · ×3  이름 · ×1      │
│                          │
│ Home Collection Binder Log│
└──────────────────────────┘
```

## Card Detail

**목적:** 카드 정보와 내 보유 상태를 확인하고 컬렉션·위시리스트를 관리한다.

**구성:** 뒤로가기, 큰 카드 이미지, 이름/세트/번호, 보유 수량·상태·메모, 행동 영역.

**핵심 액션:** 보유 카드 추가/수정, 위시리스트 토글, 바인더 배치 진입.

**상태**

- Loading: 이미지 비율과 정보 영역을 보존한다.
- Empty: 카드 정보는 있으나 미보유면 `컬렉션에 추가`를 제안한다.
- Error: 카드 정보 재시도와 목록 돌아가기를 제공한다.
- Success: 카드 이미지와 보유 상태를 명확히 구분한다.

**이동:** Collection으로 복귀하거나 Wishlist, Binder 배치 선택으로 이동한다.

```text
┌──────────────────────────┐
│ ‹ Collection        ♡    │
│       ┌──────────┐       │
│       │          │       │
│       │   CARD   │       │
│       │          │       │
│       └──────────┘       │
│ 카드 이름                 │
│ 세트 · 012/100            │
│ 보유 2장 · Near Mint       │
│ [수량/메모 수정]           │
│ [바인더에 배치]            │
└──────────────────────────┘
```

## Binder List

**목적:** 만든 바인더를 보고 새 바인더를 시작한다.

**구성:** 제목, 새 바인더 버튼, 바인더 cover/list, 카드 수와 수정일.

**핵심 액션:** 바인더 생성, 상세 열기.

**상태**

- Loading: cover 비율의 skeleton 목록을 표시한다.
- Empty: 바인더의 용도를 설명하고 첫 바인더 만들기를 제공한다.
- Error: 목록 재시도를 제공한다.
- Success: 바인더 이름, 채운 슬롯 수, 최근 수정일을 표시한다.

**이동:** 바인더 선택 시 Binder Detail로 이동한다.

```text
┌──────────────────────────┐
│ Binders              ＋  │
│ ┌──────────────────────┐ │
│ │ [cover] 최애 카드     │ │
│ │ 18 / 27 slots         │ │
│ └──────────────────────┘ │
│ ┌──────────────────────┐ │
│ │ [cover] 교환 후보     │ │
│ │ 9 / 18 slots          │ │
│ └──────────────────────┘ │
│                          │
│ Home Collection Binder Log│
└──────────────────────────┘
```

## Binder Detail

**목적:** 페이지별 카드 배치를 확인하고 슬롯을 채우거나 바꾼다.

**구성:** 뒤로가기, 바인더 이름/메뉴, 페이지 전환, 3×3 슬롯 grid, 페이지 추가.

**핵심 액션:** 빈 슬롯 채우기, 카드 보기, 교체/비우기, 페이지 이동.

**상태**

- Loading: 3×3 슬롯 크기를 유지한다.
- Empty: 첫 빈 슬롯과 카드 추가 행동을 강조한다.
- Error: 현재 페이지 번호를 유지한 채 재시도한다.
- Success: 배치된 카드가 장식보다 우선 보인다.

**이동:** 슬롯 선택 시 카드 선택 sheet 또는 Card Detail로 이동한다. drag 없이도 모든 행동이 가능해야 한다.

```text
┌──────────────────────────┐
│ ‹ Binders   최애 카드  ···│
│ Page 1 of 3        ‹  ›  │
│ [card] [card] [  + ]     │
│ [card] [  + ] [card]     │
│ [  + ] [card] [  + ]     │
│                          │
│ [       페이지 추가      ] │
│                          │
│ Home Collection Binder Log│
└──────────────────────────┘
```

## Opening Log List

**목적:** 개봉 기록을 시간순으로 보고 새 기록을 시작한다.

**구성:** 제목, 새 기록 버튼, 날짜별 기록, 대표 카드 이미지, 세트와 카드 수.

**핵심 액션:** 새 기록 작성, 기록 상세 열기.

**상태**

- Loading: 날짜와 카드 thumbnail skeleton을 표시한다.
- Empty: 첫 개봉일기 작성 행동을 제공한다.
- Error: 목록 재시도를 제공한다.
- Success: 최신 기록부터 날짜, 세트, 카드 수를 표시한다.

**이동:** `+`는 Opening Log Create, 목록 항목은 Opening Log Detail로 이동한다.

```text
┌──────────────────────────┐
│ Opening Log          ＋  │
│ JUL 16                   │
│ ┌──────────────────────┐ │
│ │[card][card] 세트명     │ │
│ │오늘의 첫 팩 · 5장       │ │
│ └──────────────────────┘ │
│ JUL 02                   │
│ [card] 세트명 · 3장       │
│                          │
│ Home Collection Binder Log│
└──────────────────────────┘
```

## Opening Log Create

**목적:** 최소 입력으로 개봉일기와 획득 카드를 저장한다.

**구성:** 취소/제목/저장, 날짜, 세트, 제목·메모, 획득 카드 선택 목록.

**핵심 액션:** 필드 입력, 카드 추가/수량 변경, 저장.

**상태**

- Loading: 카드 검색·선택 영역만 progress를 표시하고 입력값을 유지한다.
- Empty: 획득 카드가 없을 때 `카드 추가`를 보여준다.
- Error: 필드 가까이에 오류를 연결하고 서버 실패 시 입력을 보존한다.
- Success: 저장 완료 후 상세로 이동하고 중요한 완료를 알린다.

**이동:** 취소 시 목록으로, 저장 성공 시 Opening Log Detail로 이동한다.

```text
┌──────────────────────────┐
│ 취소   새 개봉일기    저장│
│ 날짜  [ 2026-07-16     ] │
│ 세트  [ 세트 선택      ›] │
│ 제목  [ 오늘의 첫 팩    ] │
│ 메모  [               ] │
│ 획득 카드                 │
│ [card] 카드 이름  − 1 ＋  │
│ [      카드 추가       ] │
└──────────────────────────┘
```

## Opening Log Detail

**목적:** 특정 개봉의 맥락과 획득 카드를 다시 본다.

**구성:** 뒤로가기, 날짜/제목/세트, 메모, 획득 카드 grid, 편집 메뉴.

**핵심 액션:** 카드 상세 열기, 기록 편집/삭제.

**상태**

- Loading: 제목과 카드 grid 비율을 유지한다.
- Empty: 획득 카드가 없는 기록이면 메모와 카드 추가 편집을 안내한다.
- Error: 재시도와 목록 복귀를 제공한다.
- Success: 기록 정보와 카드 이미지를 함께 보여준다.

**이동:** 카드 선택 시 Card Detail, 편집 시 Create와 동일한 입력 구조로 이동한다.

```text
┌──────────────────────────┐
│ ‹ Opening Log        ··· │
│ 2026. 7. 16              │
│ 오늘의 첫 팩              │
│ 세트명 · 획득 5장          │
│ "기대하던 카드를 얻었다."  │
│ [ card ]   [ card ]      │
│ [ card ]   [ card ]      │
│ [ card ]                 │
│ [        편집          ] │
└──────────────────────────┘
```

## Wishlist

**목적:** 원하는 카드를 모아 보고 컬렉션·교환 준비에 활용한다.

**구성:** 제목, 검색/필터, 카드 grid, 보유 여부와 우선순위 후보.

**핵심 액션:** 카드 상세 열기, 위시리스트 제거, 보유 카드 확인.

**상태**

- Loading: 카드 비율의 skeleton grid를 표시한다.
- Empty: 카드 상세에서 하트를 눌러 추가하는 방법을 안내한다.
- Error: 필터를 보존하고 재시도한다.
- Success: 원하는 카드와 현재 보유 여부를 색상 외 문구로 표시한다.

**이동:** 카드 선택 시 Card Detail, 향후 교환 안내에서 Trade Coming Soon으로 이동한다.

```text
┌──────────────────────────┐
│ ‹ Home       Wishlist    │
│ [ 위시 카드 검색       ] │
│ [ card ]   [ card ]      │
│ 이름 · 미보유 이름 · 1장  │
│ [ card ]   [ card ]      │
│ 이름 · 미보유 이름 · 미보유│
│                          │
│ [교환 기능 미리 보기  ›]  │
└──────────────────────────┘
```

## Trade Coming Soon

**목적:** 아직 제공하지 않는 교환 기능의 방향을 설명하고 과도한 기대를 만들지 않는다.

**구성:** 뒤로가기, 단순한 illustration 후보, 준비 중 문구, 예정 범위, 위시리스트/중복 카드 링크.

**핵심 액션:** 위시리스트 보기, 중복 카드 보기, 이전 화면 복귀.

**상태**

- Loading: 서버 데이터가 없는 정적 안내이므로 별도 loading이 필요 없다.
- Empty: 적용하지 않는다. 대신 준비 중 상태 자체가 본문이다.
- Error: 적용하지 않는다. 링크 이동 실패는 대상 화면의 error가 처리한다.
- Success: 미출시임을 분명히 밝히고 현재 가능한 준비 행동만 제공한다.

**이동:** Wishlist 또는 Collection의 중복 필터로 이동한다.

```text
┌──────────────────────────┐
│ ‹ Back                   │
│                          │
│        [cards ↔]         │
│     Trade, coming soon   │
│ 위시리스트와 중복 카드를   │
│ 바탕으로 교환을 준비해요.  │
│                          │
│ [     위시리스트 보기    ] │
│ [     중복 카드 보기     ] │
└──────────────────────────┘
```
