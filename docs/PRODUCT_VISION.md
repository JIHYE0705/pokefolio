Pokefolio는 카드 관리 앱이 아니다.

# Pokefolio 제품 비전

Pokefolio는 사용자가 포켓몬 카드를 모으며 느낀 설렘, 뿌듯함, 추억을 기록하고 언제든 다시 꺼내볼 수 있게 하는 모바일 컬렉션 저널이다.

> 언제든 내가 모은 카드와, 그 카드를 모았던 순간까지 다시 꺼내볼 수 있는 공간.

**Collect memories, not just cards.**

## 제품이 존재하는 이유

카드 한 장에는 이름과 세트 번호만 남지 않는다. 팩을 뜯기 전의 기대, 원하던 카드를 만난 순간, 여행지의 카드샵, 함께한 사람과 그날의 기분이 연결된다. 하지만 기존 도구는 보유 수량이나 시세를 정리하는 데 집중해 이 기억을 오래 보존하기 어렵다.

Pokefolio는 카드와 그 카드를 얻은 순간을 함께 기록하고, 쌓인 컬렉션을 꾸미며, 시간이 지난 뒤 자신의 수집 여정을 다시 바라볼 수 있게 하기 위해 존재한다.

## 사용자 문제

- 카드 정보는 남아도 언제, 어디서, 누구와, 어떤 기분으로 얻었는지는 쉽게 사라진다.
- 보유 목록, 위시리스트, 바인더와 개인 기록이 흩어져 하나의 수집 여정으로 이어지지 않는다.
- 기존 서비스는 시세·거래·통계를 앞세워 수집의 감정과 카드 이미지를 뒤로 밀기 쉽다.
- 기록 항목이 많으면 카드 등록이 귀찮은 데이터 입력 작업이 된다.
- 시간이 지나도 과거의 카드와 기억을 자연스럽게 다시 만날 방법이 부족하다.

## 주요 사용자

1. **추억을 남기는 컬렉터**: 팩 개봉뿐 아니라 구매, 방문, 선물, 교환과 여행의 순간을 카드와 함께 기억하고 싶다.
2. **취향을 꾸미는 컬렉터**: 색감, 포켓몬, 계절, 장소와 이야기로 바인더를 구성하고 감상하고 싶다.
3. **여정을 돌아보는 컬렉터**: 최근 카드와 과거 기록을 다시 보고 자신의 취향이 어떻게 변했는지 알고 싶다.
4. **편하게 정리하고 싶은 컬렉터**: 보유 카드, 중복과 위시리스트를 관리하되 입력 자체보다 수집의 즐거움에 집중하고 싶다.

초기 MVP는 로그인 없는 단일 사용자를 전제로 한다.

## 핵심 감정

### 설렘

새 카드와 팩을 만나고 기록을 시작하는 과정에서 다음 순간을 기대하게 한다. 과장된 애니메이션이나 보상 장치가 아니라 카드 이미지와 기록의 맥락으로 전달한다.

### 뿌듯함

쌓인 카드, 이어서 꾸미는 바인더와 변화한 취향을 보며 자신의 수집 여정을 자랑스럽게 느끼게 한다. 숫자 경쟁이나 성과 압박으로 만들지 않는다.

### 추억

카드를 획득한 날, 장소, 사람, 기분과 이야기를 함께 남기고 시간이 지난 뒤 자연스럽게 다시 보여준다.

## 제품 원칙

- **카드와 순간을 함께 기록한다.** 카드 데이터와 수집 경험을 분리된 기능으로 다루지 않는다.
- **콘텐츠가 UI보다 먼저 보인다.** 실제 카드 이미지, 사진과 짧은 기록이 장식과 통계보다 우선한다.
- **기록의 부담을 낮춘다.** 빠르게 저장할 수 있고, 원할 때만 자세히 기록한다.
- **다시 열 이유를 만든다.** 최근 카드, 이어서 꾸밀 바인더, 과거의 오늘과 작은 변화를 보여준다.
- **변화를 평가하지 않고 발견한다.** 취향과 여정은 순위나 점수보다 관찰과 이야기로 표현한다.
- **따뜻하지만 절제한다.** 즐겁고 귀여운 인상은 유지하되 유치하거나 과장된 표현을 피한다.
- **사용자가 주인공이다.** Keeper와 자동화는 사용자의 판단과 감정을 대신하지 않는다.

## 제품이 하지 않는 것

Pokefolio는 다음 제품이 되지 않는다.

- 시세와 수익률이 중심인 투자 앱
- 숫자와 통계 카드로 채운 SaaS 대시보드
- 단순한 카드 목록 CRUD 앱
- 공식 Pokémon 앱의 디자인이나 브랜드 자산을 모방한 서비스
- AI 기능을 과시하거나 매 화면에서 추천하는 앱
- 사용자의 감정보다 데이터 입력을 우선하는 관리 도구
- 복잡한 기능, 메뉴와 경쟁하는 CTA가 많은 서비스
- 카드 등록을 모든 항목 입력이 필요한 작업으로 만드는 앱

시세, 등급과 투자 정보는 향후 보조 정보가 될 수 있지만 제품의 홈, 핵심 여정과 성공 기준을 지배하지 않는다.

## 핵심 경험

```text
카드 또는 수집 경험 기록
  ├─ Collection: 내가 모은 카드와 취향을 감상
  ├─ Collection Journal: 카드를 얻은 순간과 이야기 기록
  ├─ Binder: 테마와 이야기로 카드 배치
  ├─ Memories: 과거 기록을 다시 발견
  ├─ Wishlist: 앞으로 만나고 싶은 카드 정리
  ├─ Duplicates: 여분 카드 확인
  ├─ Collection Journey: 시간에 따른 수집 변화 돌아보기
  └─ Keeper: 변화와 연결을 조용히 발견
```

사용자는 최근 카드를 다시 보고, 바인더를 이어 꾸미고, 오늘의 수집을 기록하고, 과거의 기억과 변화한 취향을 만나기 위해 Pokefolio를 다시 연다.

## Keeper의 역할

Keeper는 Pokefolio 안에서 사용자의 수집 경험을 조용히 돕는 동반자다.

Keeper는 다음을 돕는다.

- 컬렉션의 작은 변화와 취향을 관찰한다.
- 바인더와 잘 어울릴 수 있는 카드를 제안한다.
- 과거의 Collection Journal과 Memories를 다시 보여준다.
- 비슷한 분위기의 카드, 위시리스트와 중복 카드를 연결한다.
- 사용자가 작성한 기록을 짧고 자연스럽게 다듬도록 돕는다.

Keeper는 다음을 하지 않는다.

- 사용자의 감정을 임의로 단정하지 않는다.
- 구매, 투자나 교환을 강하게 권하지 않는다.
- 수치와 확률만으로 취향을 규정하지 않는다.
- 긴 분석, 명령조, 과도한 감탄사와 이모지를 사용하지 않는다.
- 관찰할 가치가 없는데도 매 화면에 등장하지 않는다.

Keeper는 관찰한 사실을 짧고 따뜻하게 말하고, 확신보다 부담 없는 제안의 형태를 사용한다.

> 최근에는 따뜻한 색감의 카드가 조금씩 많아지고 있어요.

> 봄빛 바인더에는 라티아스 AR도 잘 어울릴 것 같아요.

## 제품 판단 기준

제품과 화면을 결정할 때 다음 순서로 판단한다.

1. 사용자가 카드 또는 수집의 순간을 더 소중하게 느끼는가?
2. 기록하고 다시 보는 일이 부담 없이 이어지는가?
3. 실제 카드 이미지와 사용자의 이야기가 가장 먼저 보이는가?
4. 설렘, 뿌듯함 또는 추억 중 하나를 강화하는가?
5. 통계, 장식과 자동화가 사용자의 콘텐츠보다 앞서지 않는가?
6. 390px 모바일 환경과 접근성 요구에서 핵심 행동을 완료할 수 있는가?

## 향후 기능을 평가하는 질문

- 이 기능은 어떤 실제 사용자 순간을 더 잘 기록하거나 다시 보여주는가?
- 기능이 없어도 핵심 Collection Journal 경험은 온전히 동작하는가?
- 사용자가 입력해야 할 항목보다 얻는 감정적 가치가 큰가?
- 카드 이미지와 기억보다 숫자, 배지, 그래프 또는 AI가 더 강하게 보이지 않는가?
- 새 메뉴 없이 기존 Collection, Collection Journal, Binder, Memories 흐름에 연결할 수 있는가?
- Keeper가 제안의 근거를 짧고 이해할 수 있게 설명할 수 있는가?
- 개인정보, 사진, 장소와 함께한 사람을 사용자가 통제하고 삭제할 수 있는가?
- 실제 사용으로 필요가 확인된 기능인가, 미래를 위한 추상화인가?

하나라도 명확히 답하기 어렵다면 MVP에 추가하지 않고 사용자 검증 뒤 다시 판단한다.
