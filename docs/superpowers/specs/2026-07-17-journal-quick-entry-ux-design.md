# Pokefolio Journal 빠른 기록 UX 설계

## 목적

Pokefolio에서 첫 순간을 기록하고, 저장의 작은 보상을 느끼며, Timeline과 Home에서 기억을 다시 만나는 Journal 핵심 경험을 정의한다. 이번 설계는 제품·UX 문서만 다루며 애플리케이션 코드, Frontend 컴포넌트, Backend 모델과 API 구현을 변경하지 않는다.

## 제품 구조

Pokefolio Journal의 중심 객체는 Card가 아니라 `Moment`다.

```text
Moment
  ├─ Record type  1
  ├─ Captured at  자동
  ├─ One-line note  0..1
  ├─ Cards  0..N
  │    └─ Today's card  0..1, 연결된 Cards 중 선택
  └─ Photo  0..1 (MVP)
```

- Moment는 카드 없이도 저장할 수 있다.
- 카드샵·팝업스토어 방문, 여행과 사진만 있는 기억도 Moment다.
- Card와 Photo는 Moment에 연결되는 선택 콘텐츠다.
- Today's card는 연결된 Card가 있을 때만 선택할 수 있다.
- MVP는 사진 한 장만 지원하는 가장 작은 범위로 설계하고 여러 장 필요가 확인될 때 확장한다.

이 구조는 “Collect memories, not just cards.”를 데이터 입력 순서와 화면 위계에 적용한다.

## 빠른 기록 필드

| 필드 | 필수 여부 | 기본값·규칙 |
| --- | --- | --- |
| 기록 유형 | 필수 | 진입 경로에서 알 수 있으면 미리 설정, 일반 진입에서는 첫 단계에서 선택 |
| 날짜·시간 | 자동 | 저장 시 현재 시각, 빠른 기록 화면에서 입력받지 않음 |
| 획득 카드 | 선택 | 없음, 여러 장 선택 가능 |
| 오늘의 카드 | 선택 | 연결한 획득 카드 중 한 장 |
| 한 줄 기록 | 선택 | 없음 |
| 사진 | 선택 | 없음, MVP 한 장 |

별점, 기분, 장소, 구매 금액, 함께한 사람과 긴 본문은 MVP 빠른 기록에서 제외한다. 기록을 평가하거나 일기 form을 완성하는 부담보다 Moment를 남기는 속도를 우선한다.

## 기록 유형과 문구

기능 분류보다 사용자가 기억하는 말투를 사용한다.

| 내부 의미 후보 | 사용자 문구 |
| --- | --- |
| Pack or box opening | 📦 팩을 열었어요 |
| Card purchase or acquisition | 🛍️ 카드를 데려왔어요 |
| Gift or trade | 🎁 선물·교환이 있었어요 |
| Visit, travel or personal memory | ✨ 특별한 순간이 있었어요 |

화면 질문은 `오늘 어떤 순간을 기억하고 싶나요?`로 통일한다. `오늘 어떤 일이 있었나요?`, `카드를 샀어요`, `무엇을 남길까요?`처럼 기능적이거나 추상적인 문구는 사용하지 않는다.

사진·카드 단계는 다음 문구를 사용한다.

```text
사진이나 카드를
남겨보세요.

없어도 괜찮아요.
```

## UX flow

### 진입 경로

```text
First-run Empty의 첫 순간 기록하기
Journal Timeline의 +
Home의 오늘의 순간 기록하기
Card Detail의 이 카드와 순간 남기기
```

- 일반 진입은 기록 유형 선택부터 시작한다.
- Card Detail처럼 맥락이 있는 진입은 기록 유형과 Card를 미리 설정하고 선택 화면으로 바로 이동할 수 있다.
- 미리 설정한 값은 화면에서 확인하고 변경할 수 있다.

### 3결정 흐름

```text
1. 오늘 어떤 순간을 기억하고 싶나요?
   → 기록 유형 선택

2. 사진이나 카드를 남겨보세요.
   → 사진 찍기 또는 카드 선택
   → 없어도 건너뛸 수 있음
   → 한 줄 기록은 같은 화면의 선택 입력

3. 기억 남기기
   → 저장 중 중복 제출 방지
   → 성공 시 작은 보상 화면
```

3결정은 native 사진 picker나 카드 검색 안의 세부 조작 횟수가 아니라 사용자가 이해해야 하는 핵심 선택의 수를 뜻한다. 선택 입력을 채우지 않아도 기록 유형만으로 저장할 수 있다.

### 저장 후

저장 성공은 toast로 끝내거나 곧바로 상세 화면으로 보내지 않는다. 사용자가 방금 남긴 기억을 한 번 더 읽는 작은 보상 화면을 제공한다.

```text
오늘의 순간이 기록됐어요.

2026.07.17

“드디어 리자몽 AR!”

[컬렉션 보기]
[다른 순간 기록]
```

- 한 줄 기록이 없으면 인용문 영역을 만들지 않는다.
- `컬렉션 보기`는 연결된 카드가 있으면 Collection의 관련 카드로, 카드가 없으면 Journal Timeline으로 이동한다.
- `다른 순간 기록`은 새 일반 진입을 시작한다.
- confetti, 연속 기록, 점수와 과도한 animation을 사용하지 않는다.

```text
│                            │

│                            │
```

### Card loading

카드 선택은 전체 화면 검색으로 연다. 검색 input과 이미 선택한 Card는 유지하고 결과 영역만 loading 상태로 바꾼다.

```text
│ ‹ 돌아가기   카드 선택  완료│
│ [ 카드명·세트·번호 검색   ] │
│                            │
│ [ skeleton ] [ skeleton ]  │
│ [ skeleton ] [ skeleton ]  │
│                            │
│ 카드 불러오는 중            │
└────────────────────────────┘
```

outline, check와 접근성 선택 상태를 함께 사용한다. `완료` 후 선택 입력 화면에서 연결한 Card를 확인하고 Today's card를 선택할 수 있다.

### Today's card disabled and selected

연결한 Card가 없으면 control을 숨기지 않고 이유가 보이는 비활성 상태로 둔다. Card를 연결하면 선택 가능한 thumbnail을 보여준다.

```text
연결 Card 없음               연결 Card 있음
┌─────────────────┐         ┌─────────────────┐
│ 오늘의 카드      │         │ 오늘의 카드      │
│ 카드를 먼저      │         │ [card]✓ [card]  │
│ 추가해 주세요.   │         │ 대표 카드 선택됨 │
└─────────────────┘         └─────────────────┘
```

비활성 상태는 opacity에만 의존하지 않고 이유 문구와 semantic disabled를 사용한다. 선택 상태는 outline, check와 accessible selected state를 함께 사용한다.

## Journal Timeline

Journal Timeline은 Pokefolio의 대표 화면 후보다. Card 목록이 아니라 Moment를 시간순으로 보여준다.

```text
┌────────────────────────────┐
│ Journal                  ＋│
│                            │
│ 2026                       │
│  │                         │
│  ● 7월 17일                │
│  │ [photo] 부산 팝업        │
│  │         오늘의 카드      │
│  │         피카츄 AR        │
│  │                         │
│  ○ 7월 10일                │
│  │ 포켓몬 151 첫 개봉       │
│  │ “기대하던 날”            │
│  │                         │
│  ○ 6월 28일                │
│    여행에서 만난 카드샵      │
│    카드 없이 사진만 남긴 순간│
│                            │
└────────────────────────────┘
```

- 연도, 날짜와 세로 시간축을 기본 구조로 사용한다.
- Photo, 기록 유형, 한 줄과 Today's card 중 존재하는 콘텐츠만 표시한다.
- Card가 없는 Moment도 동일한 위계로 보여준다.
- 별점과 통계는 표시하지 않는다.
- 항목 전체가 Moment Detail 링크이며 내부에 중첩 action을 두지 않는다.

### Timeline loading

연도 제목과 시간축을 유지하고 각 Moment의 이미지·문장 영역만 skeleton으로 표시한다. 새 Moment CTA는 사용할 수 있게 둔다.

```text
┌────────────────────────────┐
│ Journal                  ＋│
│                            │
│ 2026                       │
│  │                         │
│  ● [date skeleton]         │
│  │ [image] [text lines]    │
│  │                         │
│  ○ [date skeleton]         │
│    [text lines]            │
└────────────────────────────┘
```

### Timeline empty

First-run에서는 위의 `첫 번째 추억을 만들어보세요` Empty를 사용한다. filter나 연도 조건에서 결과가 없으면 첫 사용자 Empty를 재사용하지 않고 조건을 지우는 행동을 제공한다.

### Timeline error

불러온 Moment를 숨기지 않는다. 전체 실패면 쉬운 오류 문구와 재시도, 일부 연도 실패면 해당 연도 가까이에 재시도를 둔다.

```text
┌────────────────────────────┐
│ Journal                  ＋│
│                            │
│ 기억을 불러오지 못했어요.    │
│ 잠시 후 다시 시도해 주세요.  │
│                            │
│ [        다시 불러오기     ] │
└────────────────────────────┘
```

### Timeline success

가장 최근 Moment부터 보이고 사용자가 scroll하면 과거로 이동한다. 무한 scroll, pagination과 연도 jump 방식은 데이터 양 검증 뒤 결정한다.

## Home 재방문 구조

초기 Home 우선순위는 다음과 같다.

```text
오늘 기록한 순간
→ 오늘의 카드
→ 1년 전 오늘
→ 이어서 꾸밀 Binder
```

```text
┌────────────────────────────┐
│ Pokefolio                  │
│                            │
│ 오늘 기록한 순간            │
│ [photo] 드디어 리자몽 AR!   │
│                            │
│ 오늘의 카드                 │
│ [ card ] [ card ]          │
│                            │
│ 1년 전 오늘                 │
│ 포켓몬 151 첫 개봉       ›  │
│                            │
│ 이어서 꾸미기               │
│ 봄빛 Binder          7 / 9 │
│ [   Binder 이어서 꾸미기   ]│
└────────────────────────────┘
```

- `1년 전 오늘`에 해당하는 Moment가 없으면 빈 container를 만들지 않고 section을 생략한다.
- 초기 버전에서는 Keeper를 Home에 고정 노출하지 않는다.
- 총 Card, Duplicates, Wishlist와 시세 통계는 첫 화면 우선순위에 두지 않는다.

## 휴지통과 복원

Moment Detail 전면에 `삭제`를 두지 않는다. overflow 또는 편집 하단에서 `휴지통으로 이동`을 제공한다.

```text
휴지통으로 이동
→ 이 기억을 휴지통으로 옮길까요?
→ 계속 보관 / 휴지통으로 이동
→ Timeline에서 제거
→ “휴지통으로 이동했어요. 되돌리기”
```

- 이동 직후 되돌리기를 제공한다.
- 휴지통에서 Moment preview, 이동 시각과 `복원`을 제공한다.
- 영구 삭제는 휴지통 안에서만 별도 확인 후 수행한다.
- 보관 기간과 자동 영구 삭제 여부는 정책 확정 전 구현하지 않는다.
- Moment 복원 시 연결 Card와 Photo 관계도 함께 복원되어야 한다.

```text
┌────────────────────────────┐
│ ‹ Journal        2026.07.17│
│                            │
│ [ photo / today's card ]   │
│ “드디어 리자몽 AR!”         │
│                            │
│ [           편집          ] │
│      휴지통으로 이동         │
│                            │
│ 휴지통으로 이동했어요.       │
│ 나중에 복원할 수 있어요.  되돌리기│
└────────────────────────────┘
```

휴지통 목록에서는 원래 날짜, preview와 `복원`을 우선하고 영구 삭제는 overflow 또는 별도 관리 화면 안에 둔다.

## 취소와 입력 보존

- 변경이 없으면 `취소` 또는 뒤로 가기로 즉시 닫는다.
- 입력, Card 또는 Photo 변경이 있으면 `계속 작성`과 `작성 내용 버리기`를 제공한다.
- Card 검색, Photo picker 취소는 빠른 기록 전체 취소로 취급하지 않는다.
- browser navigation과 새로고침에서 draft를 보존할지는 MVP 구현 전에 결정한다.

## 접근성 및 Apple HIG 적용

- 390px에서 한 열, content-driven height와 safe area를 사용한다.
- primary action의 touch target은 최소 44×44 CSS px를 목표로 한다.
- 기록 유형, Card와 Today's card 선택은 색상 외 check·outline·문구와 semantic state를 사용한다.
- Photo preview에는 사용자가 수정할 수 있는 대체 텍스트 정책을 후속 검토한다.
- focus는 제목 → 유형 → Photo/Card → 한 줄 → primary action 순서로 이동한다.
- validation과 저장 오류는 관련 field 또는 CTA에 연결하고 alert로 알린다.
- 200% text 확대에서 action을 가리지 않고 세로로 늘어난다.
- 저장 성공은 짧은 opacity 또는 scale 변화만 후보로 두며 `prefers-reduced-motion`에서는 즉시 전환한다.
- confetti, sparkle animation, vibration 의존과 자동 재생 motion을 사용하지 않는다.

| Save error | 모든 입력·Card·Photo | 다시 기억 남기기 |

## 검증 기준

- 첫 사용자가 데이터 부재보다 첫 추억의 가치를 먼저 읽는다.
- 일반 진입에서 유형, Photo/Card, 저장의 세 가지 핵심 결정을 이해한다.
- 기록 유형만으로 Moment를 저장할 수 있다.
- 별점과 상세 diary field가 빠른 기록에 나타나지 않는다.
- 저장 성공이 방금 남긴 기억과 다음 행동을 보여준다.
- Card 없는 Moment가 Timeline과 Home에서 깨지지 않는다.
- Home에 `1년 전 오늘`이 조건부로 나타나고 초기 Keeper는 고정 노출되지 않는다.
- 휴지통 이동 뒤 되돌리기와 복원 경로가 있다.
- loading, empty, error, saving, saved, trashed 상태가 390px와 확대 text에서 핵심 행동을 가리지 않는다.
