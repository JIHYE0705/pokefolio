# AGENTS.md

Pokefolio 저장소에서 작업하는 개발자와 자동화 에이전트가 따라야 할 지침입니다. 이 파일의 규칙은 저장소 전체에 적용합니다.

## 기본 원칙

- 요청된 범위만 변경하고 관련 없는 파일은 수정하지 않습니다.
- 기존 변경 사항을 먼저 확인하고 사용자 작업을 보존합니다.
- 기술 스택이나 의존성을 추가할 때는 목적과 영향을 문서화합니다.
- 비밀키, 인증 정보, `.env` 파일, 로컬 SQLite 데이터베이스를 커밋하지 않습니다.
- 기능 변경 시 관련 테스트와 문서를 함께 갱신합니다.

## 모노레포 경계

- `frontend/`: Next.js 애플리케이션과 프런트엔드 테스트만 둡니다.
- `backend/`: FastAPI 애플리케이션, SQLite 접근 코드, 백엔드 테스트만 둡니다.
- `docs/`: 제품 범위, 설계 결정, 로드맵 등 공용 문서를 둡니다.
- 프런트엔드와 백엔드는 공개된 API 계약을 통해 통신하며 서로의 내부 모듈을 직접 참조하지 않습니다.

## 프런트엔드 지침

- Mobile First 방식으로 작은 화면을 기본 설계합니다.
- Next.js App Router를 기준으로 라우트 구조를 구성합니다.
- 재사용 가능한 UI는 `components/`, 도메인 기능은 `features/`에 둡니다.
- 접근 가능한 시맨틱 마크업과 키보드 조작을 기본 요구사항으로 봅니다.

## 백엔드 지침

- FastAPI 엔드포인트는 `api/`, 비즈니스 규칙은 `services/`에 분리합니다.
- SQLite 접근은 `repositories/`와 `db/` 경계를 통해 수행합니다.
- 외부 입력과 API 출력은 명시적인 스키마로 검증합니다.
- API 계층에 데이터베이스 쿼리나 핵심 비즈니스 로직을 직접 작성하지 않습니다.

## 검증 원칙

- 작업 완료 전 변경 파일과 Git 상태를 확인합니다.
- 구현 단계에서는 프런트엔드와 백엔드 테스트를 각각 실행합니다.
- 실행할 수 없는 검증이 있다면 완료 보고에 이유를 명시합니다.

## 현재 단계

현재는 구조 및 문서 초기화 단계입니다. 별도 요청 전까지 구현 코드, 패키지 설정, 데이터베이스 또는 의존성을 추가하지 않습니다.
